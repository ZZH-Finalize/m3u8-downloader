# 编码器选择指南

在添加任务或系统设置中，您可以选择输出编码格式和编码器类型。合理选择编码器可以显著提升转码效率和质量。

## 部署环境说明

### Docker 镜像用户
 
如果您使用项目提供的 Docker 镜像运行后端服务，请注意：
 
Docker 镜像内的编码器组件已全部移除，使用 Docker 镜像后端时，用户只能选择 copy 作为编码方式，copy 是最常用的模式，它直接复制视频流而不重新编码。
 
**注意事项：**
 
- Docker 容器默认无法访问宿主机的 GPU 设备
- 如果您确实需要其他编码方式，可以使用原生部署直接使用宿主机上功能齐全的 ffmpeg，或者自行转码处理 Docker 合并完成的输出文件

### 宿主机部署用户

如果您直接在宿主机上部署 Python 脚本和 FFmpeg，编码器的可用性取决于：

1. **宿主机的操作系统**（Windows、Linux、macOS）
2. **FFmpeg 的编译选项**（是否启用了特定编码器）
3. **硬件设备**（是否有对应的 GPU）
4. **驱动程序**（是否安装了正确的驱动）

请根据下文来确认您的环境支持的编码器。

---

## 检查 FFmpeg 支持的编码器

在选择编码器之前，建议先检查后端 FFmpeg 支持的编码器列表。在命令行运行：

```bash
ffmpeg -encoders
```

或者查看特定编码器的支持情况：

```bash
ffmpeg -encoders | grep -E "nvenc|qsv|amf|vaapi|mf|d3d12va|vulkan|h264|hevc|av1"
```

## 编码器选择建议

### 1. 优先选择：复制流（copy）

**没有特殊需求时，强烈建议选择 `copy`（复制流）模式。**

#### 优点

- 不重新编码，直接复制视频流，速度最快
- 无画质损失，保持原始质量
- CPU/GPU 占用最低
- 编码器设置不生效，无需关心编码器类型

#### 适用场景

- 源视频编码格式与目标设备兼容
- 追求最快速度下载
- 不需要改变视频编码格式

---

### 2. 需要转码时：硬件编码器（优先推荐）

如果必须改变视频编码格式，优先选择硬件编码器：

| 编码器 | 厂商/平台 | 适用硬件 |
|--------|----------|---------|
| `nvenc` | NVIDIA | NVIDIA GPU |
| `amf` | AMD | AMD GPU |
| `qsv` | Intel | Intel 核显/独显 |

#### 优点

- 利用专用硬件编码单元，速度快
- CPU 占用低，可并行处理其他任务
- 功耗相对较低

#### 注意事项

- 需要对应厂商的显卡和驱动程序
- 部分老旧硬件可能不支持新编码格式（如 AV1）

---

### 3. 操作系统接口编码器（次选推荐）

如果硬件编码器不可用，可选择操作系统提供的媒体加速接口：

| 编码器 | 平台 | 适用系统 | 说明 |
|--------|------|---------|------|
| `mf` | Windows | Windows | Windows Media Foundation，Windows 系统推荐 |
| `d3d12va` | Windows | Windows 10/11 | Direct3D 12 Video Acceleration，更现代的 Windows 接口 |
| `vaapi` | Linux | Linux | VAAPI 接口，Linux 系统常用 |

#### 优点

- 系统集成度高，兼容性好
- 支持多种编码格式
- 同样能够支持硬件编码，但能否成功调用硬件取决于操作系统

#### 注意事项

- 需要系统正确配置媒体加速支持
- Linux 系统可能需要安装额外驱动包

---

### 4. Vulkan 编码器（备选）

#### 适用场景

- 跨平台需求
- 其他编码器不可用时的备选方案

#### 特点

- 基于 Vulkan API，跨平台支持好
- 性能因实现而异

---

### 5. 软件编码（最后选择）

**仅在硬件编码器不可用时使用。**

- **编码器**：`software`

#### 特点

- 使用 CPU 进行编码，兼容性最好
- 速度最慢，CPU 占用高
- 编码质量可控，但耗时长

#### 适用场景

- 没有可用的硬件编码器
- 对编码质量有极高要求，愿意牺牲时间
- 调试和测试用途


## 常见问题

### Q: 选择了硬件编码器但报错？

**A:** 可能原因：

1. 硬件不支持该编码格式（如老旧显卡不支持 AV1）
2. 驱动程序过旧，请更新显卡驱动
3. FFmpeg 编译时未启用该编码器

### Q: 如何确认编码器是否可用？

**A:** 使用以下命令测试：

```bash
ffmpeg -f lavfi -i testsrc=duration=1:size=320x240:rate=30 -c:v <编码器名称> -f null -
```

例如测试 nvenc：

```bash
ffmpeg -f lavfi -i testsrc=duration=1:size=320x240:rate=30 -c:v h264_nvenc -f null -
```

如果命令执行成功且无错误输出，说明该编码器可用。

### Q: copy 模式下编码器设置会生效吗？

**A:** 不会。当输出编码选择 `copy` 时，编码器设置被忽略，视频流直接复制不重新编码。

### Q: 如何选择合适的输出编码格式？

**A:** 根据目标设备和播放平台选择：

| 编码格式 | 兼容性 | 压缩效率 | 推荐用途 |
|---------|--------|---------|---------|
| `h264` | 最好 | 一般 | 通用兼容，老设备播放 |
| `hevc` (H.265) | 较好 | 高 | 高清视频，节省空间 |
| `av1` | 一般（新设备支持） | 最高 | 最新设备，网络传输 |
| `copy` | 取决于源视频 | 无损失 | 保持原画质，最快速度 |
